# /ac-init {描述?}

## 输入规则

- 描述为可选参数，用于补充项目背景、技术栈、特殊约束。

## 执行步骤

1. **判断项目类型**：检测项目根目录是否存在有效代码文件和目录结构。
   - 新项目：无实质代码，或用户明确说明是新项目。
   - 旧项目：已有代码、配置文件和目录结构。

2. **处理 AGENTS.md**：
   - 新项目：
     1. 通过提问澄清项目需求、技术栈、开发规范。
     2. 生成高质量 `AGENTS.md`（须满足下方质量标准）。
     3. 继续执行 `ac-plan` 创建初始计划。
   - 旧项目：
     1. 若根目录不存在 `AGENTS.md`：通过提问与代码阅读收集信息后生成。
     2. 若已存在 `AGENTS.md`：按质量标准评估，不足时增补优化。
     3. 默认不创建计划（除非用户明确要求）。

## 高质量 AGENTS.md 标准

生成或优化 `AGENTS.md` 时必须满足以下检查清单：

- [ ] 常用命令与约束前置，降低代理执行歧义。
- [ ] 明确列出目录结构（至少到第二层）。
- [ ] 明确标注技术栈及版本。
- [ ] 明确代码风格约束（命名规范、格式化工具、lint 配置）。
- [ ] 内容精简，无冗余段落或模板化填充。
- [ ] 大型单体仓库按子包拆分维护本地 `AGENTS.md`。

## 输出要求

- 完成后向用户反馈：项目类型判定结果、AGENTS.md 处理结果、是否需要后续 `ac-plan`。

## 失败条件

- 无法判断项目类型时 → 向用户提问澄清，不可假设。
- AGENTS.md 生成后不满足质量标准 → 重新优化直至满足。

## 完成提示

执行完毕后，必须使用以下固定格式提示用户下一步操作：

> **下一步建议：**
> - `/ac-plan`：创建新计划
