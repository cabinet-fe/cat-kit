# 增加执行纪律约束

## 补丁内容

在 `renderNavigator()` 生成的 SKILL.md 中新增「执行纪律」节（`## 执行纪律`），位于标题描述与意图匹配表之间，用 `<CRITICAL>` 标签包裹四条强制规则：

1. **协议先行**：匹配到动作后必须先读取对应 `actions/*.md` 协议文件的完整内容，禁止凭记忆或猜测跳过。
2. **前置检查必做**：所有动作的前置检查必须逐条执行，包括 `agent-context validate`。
3. **禁止直接改动**：未经 plan/rush 创建计划，不得直接修改项目代码文件；任何变更必须在 implement 或 patch 上下文中进行。
4. **顺序执行**：协议步骤按编号逐项执行，不可跳步、合并或并行。

同时将标题描述从"严格执行"强化为"**必须先读取对应协议文件的完整内容**，再严格按协议步骤逐项执行"。

**修改原因**：代理在实际使用 ac-workflow 时，常跳过协议文件的读取步骤，直接根据用户意图寻找并修改文件，未遵循工作流的前置检查和步骤顺序。

## 影响范围

- 修改文件: `packages/agent-context/src/content/index.ts`
- 修改文件: `.cursor/skills/ac-workflow/SKILL.md`
