# /ac-plan - 创建新计划

用法：`/ac-plan {简短需求描述}`

该命令用于在维护阶段创建一个新活跃计划。描述较短时，允许多轮澄清后再落盘。

## 严格前置验证（必须全部通过才能继续）

### ✅ 验证 1：活跃计划数量必须为 0

- 扫描 `.plan/` 下的 `plan-{number}` 目录（不包括 `backlog/`、`done/`）。
- 若数量为 1，终止并提示：

  > ❌ 当前已有活跃计划 plan-{number}，不能新增计划。请先完成并归档后再创建。

- 若数量大于 1，终止并提示：

  > ❌ 检测到多个活跃计划，工作流已异常。请先恢复到单活跃状态。

### ✅ 验证 2：backlog 必须为空

- 扫描 `.plan/backlog/` 下的 `plan-{number}` 目录。
- 若存在任意 backlog 计划，终止并提示：

  > ❌ 当前 backlog 中仍有待执行计划，请先完成队列中的计划，暂不允许新增。

## 执行步骤

1. **确定新计划编号**：
   - 同时扫描：
     - `.plan/plan-{number}`
     - `.plan/backlog/plan-{number}`
     - `.plan/done/plan-{number}-{date}`
   - 提取所有编号，取最大值 + 1。
   - 若不存在任何历史编号，使用 `1`。

2. **理解需求并澄清**：
   - 阅读用户描述。
   - 若目标/范围/验收不明确，先澄清，不做主观假设。

3. **创建目录与文件**：
   - 创建 `.plan/plan-{新编号}/plan.md`（活跃计划）。
   - 使用模板：

   ```markdown
   # {计划名称}

   > 状态: 未执行

   ## 目标

   {明确的目标描述}

   ## 内容

   {详细的实施步骤和内容}

   ## 影响范围

   ## 历史补丁
   ```

4. **告知用户**：输出新计划编号与摘要，提示可执行 `/ac-implement`。

## 注意事项

- 计划编号必须全局递增，不复用。
- `## 目标` 与 `## 内容` 必须是实质内容，不能仅保留空标题。
