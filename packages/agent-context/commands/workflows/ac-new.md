# /ac-new - 创建全新项目

该命令用于在全新项目中初始化计划系统。允许一次创建多个计划：首个计划为活跃计划，其余进入 backlog。

## 严格前置验证（必须全部通过才能继续）

### ✅ 验证 1：必须是全新项目

- 检查项目根目录是否已存在业务代码或常见项目入口（如 `src/`、`app/`、`package.json`、`index.html` 等）。
- 检查是否已存在：
  - `.plan/plan-{number}`
  - `.plan/backlog/plan-{number}`
  - `.plan/done/plan-{number}-{date}`
- **如果以上任一存在 → 终止执行**，告知用户：

  > ❌ 当前项目不是全新项目，无法使用 /ac-new。请使用 /ac-plan 创建新计划。

## 执行步骤

1. **与用户沟通**：确认项目目标、技术栈与初始计划拆分。
2. **拆分计划**：根据复杂度拆分为 `plan-1 ... plan-n`（`n >= 1`）。
3. **创建目录结构**：

   ```
   .plan/
     done/
     backlog/
     plan-1/
       plan.md
   ```

4. **生成计划文件**：
   - `plan-1` 创建在 `.plan/plan-1/plan.md`（活跃计划）。
   - `plan-2 ... plan-n` 创建在 `.plan/backlog/plan-{number}/plan.md`（待执行队列）。

5. **`plan.md` 模板**（每个计划都一致）：

   ```markdown
   # {计划名称}

   > 状态: 未执行

   ## 目标

   {明确的目标描述}

   ## 内容

   {详细的实施步骤和内容}

   ## 影响范围

   ## 历史补丁
   ```

6. **告知用户**：输出初始化结果（活跃计划 + backlog 列表），提示下一步使用 `/ac-implement` 执行当前活跃计划。

## 注意事项

- `/ac-new` 只负责初始化计划，不执行实现。
- 初始化阶段允许多计划；维护阶段仍遵循“单活跃计划模型”。
