# /ac-implement - 实现当前计划

该命令用于实现当前活跃计划。执行完成表示“本轮实现已完成”，不等于“计划目标完全达成”。

## 严格前置验证（必须全部通过才能继续）

### ✅ 验证 1：必须存在且仅存在一个活跃计划

- 扫描 `.plan/` 下 `plan-{number}` 目录（不包括 `.plan/backlog/` 与 `.plan/done/`）。
- 若数量为 0，终止并提示：

  > ❌ 当前没有待执行计划，请先使用 /ac-plan 创建计划。

- 若数量大于 1，终止并提示：

  > ❌ 检测到多个活跃计划，工作流已异常。请先恢复为单活跃状态。

### ✅ 验证 2：当前计划状态必须为 `未执行`

- 读取当前 `plan.md`。
- 必须包含 `> 状态: 未执行`。
- 若已是 `> 状态: 已执行`，终止并提示：

  > ❌ 当前计划已执行过，请使用 /ac-patch 继续迭代，或使用 /ac-done 归档。

### ✅ 验证 3：计划内容有效

- `## 目标` 与 `## 内容` 必须有实质内容。
- 仅占位文本（如“待补充”“TODO”）视为无效。

## 执行步骤

1. **分析现状**：理解项目结构、依赖与约束。
2. **按计划实施**：依据 `## 内容` 完成开发。
3. **验证实现**：构建/运行/测试通过，且满足 `## 目标`。
4. **更新 `plan.md`**：
   - 将 `> 状态: 未执行` 改为 `> 状态: 已执行`。
   - 在 `## 影响范围` 中记录本次新增/修改/删除文件，例如：

   ```markdown
   - 新增文件: `/path/to/new-file`
   - 修改文件: `/path/to/modified-file`
   - 删除文件: `/path/to/deleted-file`
   ```

5. **告知用户**：输出执行结果，并提示后续可用 `/ac-patch` 或 `/ac-done`。

## 注意事项

- `/ac-implement` 仅操作当前活跃计划，不直接操作 backlog 计划。
- 状态必须在执行完成后再变更为 `已执行`。
- 影响范围必须完整且准确。
