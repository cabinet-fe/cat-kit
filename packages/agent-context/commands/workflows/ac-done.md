# /ac-done - 完成并归档当前计划

该命令用于将当前活跃计划标记为真正完成并归档。归档后会自动晋升 backlog 中编号最小的计划为新的活跃计划（如存在）。

## 严格前置验证（必须全部通过才能继续）

### ✅ 验证 1：必须存在且仅存在一个活跃计划

- 扫描 `.plan/` 下 `plan-{number}` 目录（不包括 `.plan/backlog/` 与 `.plan/done/`）。
- 若数量为 0，终止并提示：

  > ❌ 当前没有可归档的活跃计划。

- 若数量大于 1，终止并提示：

  > ❌ 检测到多个活跃计划，工作流已异常。请先恢复为单活跃状态。

### ✅ 验证 2：当前计划状态必须为 `已执行`

- 读取当前 `plan.md`，必须包含 `> 状态: 已执行`。
- 若不是 `已执行`，终止并提示：

  > ❌ 当前计划尚未执行完成，请先使用 /ac-implement。

### ✅ 验证 3：必须得到用户确认“任务真正完成”

- 若用户未确认完成，终止并建议继续使用 `/ac-patch` 迭代。

## 执行步骤

1. **获取当前日期**：使用 `YYYYMMDD` 格式（例如 `20260304`）。
2. **执行归档移动**：

   ```bash
   mv .plan/plan-{number} .plan/done/plan-{number}-{date}
   ```

3. **自动晋升 backlog 计划**：
   - 扫描 `.plan/backlog/` 下合法目录 `plan-{number}`。
   - 若存在多个，选择编号最小者。
   - 将其移动为新的活跃计划：

   ```bash
   mv .plan/backlog/plan-{next_number} .plan/plan-{next_number}
   ```

4. **校验结果**：
   - 归档目录存在：`.plan/done/plan-{number}-{date}/`。
   - 原活跃目录不存在：`.plan/plan-{number}/`。
   - 若有晋升，新的 `.plan/plan-{next_number}/` 必须存在。

5. **告知用户**：输出归档信息，以及是否已自动晋升下一个活跃计划。

## 注意事项

- 归档后计划编号不复用。
- 日期必须使用系统当天日期，格式固定 `YYYYMMDD`。
- 若 backlog 为空，则归档后进入“无活跃计划”状态。
