# /ac-init

## 输入规则

- 可附带描述，用于补充项目背景、技术栈、约束。

## 执行规范

1. 识别项目是新项目还是旧项目。
2. 新项目：先澄清需求并生成高质量 `AGENTS.md`，然后继续执行 `ac-plan`。
3. 旧项目：
- 若根目录不存在 `AGENTS.md`，通过提问与代码阅读补全并生成。
- 若已存在 `AGENTS.md`，先评估质量，不足时增补优化。
4. 区分原则：新项目创建计划；旧项目默认不创建计划。

## 高质量 `AGENTS.md` 标准

- 常用命令与约束前置。
- 明确目录结构、技术栈与代码风格约束。
- 内容精简，避免冗余上下文。
- 大型单体仓库可按子包拆分维护本地 `AGENTS.md`。
