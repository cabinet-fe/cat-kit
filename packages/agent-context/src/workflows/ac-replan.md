# /ac-replan {描述}

## 严格前置验证

- 必须附带描述。
- 至少存在一个未实施计划（`preparing` 或当前未实施计划）。
- 默认作用域为 `.agent-context/preparing` 中全部未实施计划；可指定仅重规划部分计划。
- 若当前计划为 `已执行`，禁止重写当前计划内容，仅允许重规划 `preparing` 队列。
- 若当前计划为 `未执行` 且用户明确要求，可将当前计划纳入重规划范围。
- 若检测到多个当前计划，拒绝执行并提示先恢复单活跃状态。

## 冲突处理

- 目标计划不存在：拒绝执行并列出可选计划编号。
- 目标计划已执行：拒绝执行并提示使用 `ac-patch`。

## 执行结果约束

- 重规划后必须保持“单当前计划 + 若干 preparing 计划”。
- 新增计划编号继续使用全局最大编号 + 1。
- 已有未改动计划保持原编号。
