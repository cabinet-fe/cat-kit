# /ac-patch {描述}

## 严格前置验证

- 必须附带描述。
- 必须存在且仅存在一个当前计划。
- 当前计划状态必须为 `已执行`。

## 执行步骤

1. 扫描当前计划目录下 `patch-{number}.md`，取最大编号 + 1。
2. 阅读 `plan.md` 与历史补丁，避免重复修复。
3. 执行补丁并完成验证。
4. 生成 `patch-{number}.md`，记录补丁内容与影响范围。
5. 回写 `plan.md`：
- 在 `## 历史补丁` 追加补丁记录（去重）。
- 合并更新 `## 影响范围`（去重）。

## 状态约束

- `ac-patch` 不改变计划状态，状态保持 `已执行`。
