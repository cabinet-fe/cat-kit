# /ac-implement

## 严格前置验证

- 不可附带描述。
- 必须存在且仅存在一个当前计划。
- 当前计划状态必须为 `未执行`。
- `plan.md` 的 `## 目标` 与 `## 内容` 需有实质内容。

## 执行步骤

1. 依据 `plan.md` 中 `## 内容` 实施当前计划。
2. 完成必要验证后，将状态行更新为 `> 状态: 已执行`。
3. 更新 `plan.md` 的 `## 影响范围`，记录新增/修改/删除文件。

## 注意事项

- `ac-implement` 仅操作当前计划，不直接操作 preparing 计划。
- 状态更新必须发生在实施完成之后。
