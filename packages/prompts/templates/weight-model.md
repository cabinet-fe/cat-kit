# 开发权重模型

当需要在多个关注点之间做出权衡时，按以下优先级顺序考虑：

| 优先级 | 关注点 | 说明 |
|--------|--------|------|
| 1 | 正确性 | 代码必须正确实现功能需求 |
| 2 | 安全性 | 防止安全漏洞和数据泄露 |
| 3 | 性能 | 满足性能需求，避免明显瓶颈 |
| 4 | 可扩展性 | 代码易于修改和扩展 |
| 5 | 前瞻性 | 为未来可能的需求预留空间 |

## 常见权衡场景

### 简单实现 vs 可扩展设计

- **选择简单实现**：需求明确且不太可能变化
- **选择可扩展设计**：有明确的扩展计划

### 性能优化 vs 代码可读性

- **优先可读性**：非热点代码
- **优先性能**：已确认的性能瓶颈
